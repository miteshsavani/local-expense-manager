# PROJECT CODING INSTRUCTIONS (UI SYSTEM)

## ARCHITECTURE RULES

- CSS is split into:
  base / layout / components / views
- JS is split into:
  core / services / managers / views / components / utils
- HTML is modular under components/html

---

## FILE SELECTION RULE

Always follow routing-map.json or routing-map.md:

### UI changes → CSS first
### Behavior changes → JS first
### Screen-level logic → views/
### Reusable UI → components/
### API / sync / storage → services/

---

## CSS RULES

- Never mix layout + component styles
- Views CSS = screen-specific only
- Components CSS = reusable UI only
- Layout CSS = structure only

---

## JS RULES

- Managers = UI + domain orchestration
- Services = external systems (firebase, sync, network)
- Utils = pure functions only
- Views = render logic only
- Components = UI widgets only

---

## PERFORMANCE RULES

- Avoid duplicate DOM queries
- Prefer event delegation
- Avoid inline styles
- Keep bootstrap.js minimal

---

## DEBUG RULE

If UI breaks:
1. check views/
2. check components/
3. check layout/
4. check managers/ui.js

---

## AGENT RULE

Always use routing-map.json before making file edits.
If uncertain → ask before modifying multiple layers.
